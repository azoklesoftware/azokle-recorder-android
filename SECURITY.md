# Security Policy

## Supported Versions

| Version | Supported |
|---|---|
| Latest release | ✅ Yes |
| Previous major | ⚠️ Critical fixes only |
| Older versions | ❌ No |

---

## Reporting a Vulnerability

**Do not report security vulnerabilities through public GitHub issues.**

Email us at: **📧 security@azokle.com**

Please include:
- Description of the vulnerability and its potential impact
- Steps to reproduce (device model, Android version, app version)
- Proof of concept or exploit code (if available)
- Suggested fix (optional)

### Response Timeline

| Step | Timeline |
|---|---|
| Acknowledgement | Within 48 hours |
| Initial triage | Within 5 business days |
| Status update | Within 14 days |
| Patch release | Within 90 days of confirmation |

We will coordinate disclosure timing and credit you in the release notes (unless you prefer anonymity).

---

## Scope

**In scope:**
- Audio data leakage or unauthorised access
- Permission bypass vulnerabilities
- Intent redirection / interception attacks
- Insecure data storage
- Denial-of-service in the recording service

**Out of scope:**
- Issues in third-party libraries (report upstream)
- Vulnerabilities requiring physical access to an unlocked device
- Issues on unofficial/modified Android builds

---

## Our Commitment

- Respond promptly and transparently
- Not pursue legal action against researchers acting in good faith
- Credit researchers in our changelog (with permission)
